## вңҰ HTTPS Tunnel (Cloudflare)

The bot uses **pycloudflared** вҖ” a Python package that downloads the `cloudflared` binary automatically on first run. No CLI installs, no system packages вҖ” works on Pterodactyl and any Python host.

**Why Cloudflare over ngrok:**
- вң… Unlimited bandwidth & requests вҖ” no monthly caps
- вң… Permanent URL that never changes between restarts
- вң… Free вҖ” no paid plan needed
- вң… Zero system installs вҖ” binary downloads via Python

**Setup (browser only, no CLI needed):**

1. Go to [one.dash.cloudflare.com](https://one.dash.cloudflare.com) вҶ’ **Networks вҶ’ Tunnels вҶ’ Create a tunnel**
2. Choose **Cloudflared**, give it a name (e.g. `nyzro-api`), save
3. On the **Install connector** step, copy the token from the command shown:
   ```
   cloudflared tunnel run --token <COPY_THIS>
   ```
4. Go to **Public Hostname** tab вҶ’ add a hostname:
   - Subdomain: `api` В· Domain: `yourdomain.com` В· Service: `http://localhost:8000`
5. Add to `bot/.env`:
   ```env
   CF_TUNNEL_TOKEN = "eyJhIjoiXXXX..."
   CF_TUNNEL_URL   = "https://api.yourdomain.com"
   ```

On every startup the console prints:
```
в—Ҳ Tunnel: cloudflared binary ready вҖ” starting tunnel on port 8000вҖҰ
в—Ҳ Tunnel: API is live at  https://api.yourdomain.com
  вҶі NEXT_PUBLIC_API_URL = https://api.yourdomain.com/api/v1
```

## вңҰ Emoji Sync

Runs automatically on startup when `EMOJI_SYNC=true`:

```
вҳ… Starting Application Emoji Sync вҖ” 144 unique emojis found in emoji.py
в—Ҳ Found 144 templates | Application hosts 202 emojis
вҶ‘ Uploading: ztick  (not in application emojis)
вң” Uploaded: ztick  [saved as ID: 1234567890]
вң” emoji.py patched in-place to reflect current API state.
вҳ… Restarting bot to load updated emoji IDs...
```

| Event | Action |
|---|---|
| New emoji found | Uploaded to application, ID written to `emoji.py` |
| Stale ID detected | `emoji.py` patched automatically |
| No changes | Sync completes instantly, no restart |
| After any patch | Bot restarts itself so fresh IDs are live |

## вңҰ Troubleshooting

| Problem | Fix |
|---|---|
| Bot fails to start | Check `TOKEN` is set and bot has correct gateway intents |
| Music not working | Verify `LAVALINK_HOST`, `LAVALINK_SECURE`, and `LAVALINK_PORT` |
| Dashboard auth error | Check Discord OAuth client ID/secret and redirect URI |
| Dashboard can't load data | Confirm `API_ENABLED=true`, bot is running, `NEXT_PUBLIC_API_URL` is correct |
| Emojis showing as plain text | Run with `EMOJI_SYNC=true` once to upload and patch IDs |
| CORS errors from dashboard | Add your Vercel URL to `CORS_ORIGINS` in `bot/.env` |
| Tunnel not starting | Check `CF_TUNNEL_TOKEN` is valid and `pycloudflared` is installed |
| Tunnel URL changed | Set `CF_TUNNEL_URL` вҖ” named tunnels always produce the same URL |
